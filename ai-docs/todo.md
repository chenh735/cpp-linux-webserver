## 需要优先修复

- [x] 修正 HTTP 响应头字段名。
  - 当前代码中写的是 `Connect: close`。
  - 正确字段应为 `Connection: close`。
  - 这个字段用于告诉客户端响应结束后服务端会关闭连接。

- [x] 调整 `recv` 失败时的控制流。
  - 当前 `recv == -1` 后会 `break`，这会跳出整个服务器主循环。
  - 更合理的行为是关闭当前 `client_fd`，然后继续等待下一个连接。

- [x] 调整 `send == 0` 时的处理方式。
  - 当前逻辑中关闭 `client_fd` 后使用 `continue`，容易让循环继续对已关闭 fd 发送数据。
  - 更合理的行为是关闭连接并退出当前发送循环。

- [x] 清理 `perror` 的使用方式。
  - `perror` 适合打印系统调用失败原因。
  - 客户端正常关闭连接时，`recv == 0` 不是错误，不建议用 `perror`。
  - `perror` 的提示字符串不要带换行。

- [x] 修复源码中的中文乱码问题。
  - 统一保存为 UTF-8。
  - 如果终端环境显示中文不稳定，可以先用英文错误提示。

## 建议优化

- [x] 校验端口范围时排除非预期端口。
  - 当前允许 `0`，这会让系统自动分配端口。
  - 学习阶段建议限制为 `1-65535`，更符合手动指定端口的预期。

- [ ] 在服务器启动时打印监听端口。
  - 示例：`server listening on port 8080`。
  - 这样便于确认程序是否按预期启动。

- [ ] 将单个客户端连接处理逻辑提取成函数。
  - 例如 `handle_client(int client_fd)`。
  - 这样主循环会更清楚：`accept` 成功后只负责分发处理。

- [ ] 将创建监听 socket 的逻辑提取成函数。
  - 例如 `create_listen_socket(uint16_t port)`。
  - 便于后续加入非阻塞设置、错误处理和配置项。

- [ ] 补充简单测试记录。
  - 使用浏览器访问。
  - 使用 `curl -v http://127.0.0.1:8080` 查看响应头。
  - 多次刷新确认 fd 不泄漏、服务不会退出。
